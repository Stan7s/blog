Obsidian应用指南
2022-01-02 15:13:31 # 软件配置 # 笔记软件 # 双向链接 # Wiki

Welcome to my Vault! 👻👻

像年度总结里提到的那样，新年是在Obsidian的“地窖 (Vault)”（相当于OneNote里的笔记本）里度过的。对于双链笔记的讨论网上已经有很多。对目前仅写了几十个词条的我而言，它的作用还远没有到“激发新灵感”那一步。我单纯地享受构造的过程（不学工程真是可惜了x）。

相对于其他笔记应用，Obsidian最大的优点就是抗造。之前深度使用过思源笔记，它早期有段时间markdown语法渲染问题挺多，经常出现排版混乱；笔记数量多到一定程度会卡顿；最头疼的是每次更新时需要备份并创建新文件夹，强迫症如我为了不更改路径只能先挪出来再挪回去。当然按照作者的迭代速度，这些现在很可能都已经修复了。获得一致好评的Notion像精致餐厅，服务无懈可击，标签颜色像寿司一样一个个给你设计好；但侍者面带标准微笑，一句话都不会跟你多聊，你也不知道他在想什么（存储基于云端）。更早一些的笔记应用（为知笔记、印象笔记、OneNote、幕布）更是连API都很少或干脆没有。Obsidian则完全基于本地markdown，不（主动）同步也不加密，主题设置里直接鼓励你上手改CSS，即时渲染也没有出现过错误，除了中文输入法导致的markdown语法输入有点小问题（例如输入一个*会出现三个***）之外，可以做到无痛排版。

Obsidian对自定义的高度支持也满足了软件爱好者的探索欲。例如在开启vim语法支持时，它会告诉你“vim很危险，请输入exit without saving的命令来证明你不是小白”。在开启Community Plugins时，会提示“第三方插件可能偷看你的照片，阅读你的邮件，你确定要关闭Safe Mode吗？”尽管社区的活跃生态明明离不开官方的鼓励。这种“先吓唬你一下，之后随便你折腾”且折腾之后不出bug的体验还是很有趣的。

基本使用方法就不介绍了，参见Obsidian中文教程。下面是一些个人使用技巧。

同步篇

Windows和iOS同步

指南

Obsidian Help里这样描述iOS同步功能：

If you already have a vault on your desktop, here’s the two ways to sync your vault to mobile and access it:

  • iCloud Drive Sync
  • Obsidian Sync

The two current officially supported sync solutions are iCloud and Obsidian Sync.
Working Copy (git) is another alternative that has been tested to work with Obsidian on iOS.

Currently, there is no known support for the following sync services on iOS yet:

  • Dropbox
  • Google Drive
  • OneDrive
  • Syncthing

If you find out a way to sync using these services on iOS with Obsidian, we would be glad if you can join our Discord server and teach us how to set it up.

后面还有一段在解释为什么不支持绝大多数第三方同步。核心意思是说，Obsidian的工作基于整个Vault而不是单个文档，当一个文档被更改，Vault中其他相关的一堆文件 (反向链接、图片嵌入、自动补全等）都会跟着更改。大多数markdown编辑器只针对单个文档，而第三方同步服务往往也只提供单个文档的下载、编辑和上传的API。Obsidian的每次编辑都要update几乎整个Vault，由此导致的延迟会引发很多错误。有一些同步服务可以通过第三方App设置实现（例如DropSync和FolderSync），但由于iOS的app sandboxing，iPhone用户就不要想了。

下面记录我通过iCloud实现桌面端和手机端的Obsidian同步的过程。

初始状态

  • 桌面端 (Windows)：一个Obsidian App和一个存储笔记的文件夹（假设路径为path\to\my_obsidian\my_vault）。尚未安装iCloud App.
  • 手机端 (iOS)：一个刚安装还没打开过的Obsidian App.

警告

开始操作前务必备份my_obsidian文件夹。

步骤

  1. 在桌面端安装iCloud App。云盘默认路径是C:\Users\{your_username}\iCloudDrive，且不易更改（🍎真有你的）。
  2. 在手机端打开Obsidian App，选择 Create New Vault，随便输一个名称（例如my_vault_2，但不重要），勾选iCloud同步。
  3. 此时桌面端的云盘路径下应该有了Obsidian创建的文件夹。注意文件夹名称很可能不是Obsidian而是iCloud~md~obsidian。完整路径即C:\Users\{your_username}\iCloudDrive\iCloud~md~obsidian。
  4. 在桌面端将my_vault文件夹复制到C:\Users\{your_username}\iCloudDrive\iCloud~md~obsidian路径下。
  5. 在手机端打开Command Palette，搜索并点击 “Open Another Vault”，选择my_vault。熟悉的地窖出现了！
  6. 删除旧路径下的my_obsidian文件夹。

至此已经实现了桌面端和手机端的同步。注意此时桌面端的笔记存储路径变成了iCloud云盘，之前的路径失效了。如果想更新到云盘下的路径，需要在桌面端Obsidian App重新打开云盘下的my_vault文件夹。

如果想仍然想沿用之前的路径，可以继续以下操作（有风险且没必要，如果有步骤不理解的话不建议操作）：

  1. 在桌面端打开cmd，执行mklink -J "path\to\my_obsidian" "C:\Users\{your_username}\iCloudDrive\iCloud~md~obsidian"。注意：1) mklink是cmd的内置命令，因此必须在cmd（而不是Powershell）执行。2) 由于mklink [[/d] | [/h] | [/j]] <link> <target> 是将 target 映射到新创建的 link，因此必须保证 link 的路径不存在（即必须执行步骤6）。mklink的文档见这里。
  2. 此时path\to\my_obsidian\my_vault重新出现。
  3. 在桌面端打开Obsidian App，能正常打开说明映射成功。

【2022-01-05】更新：iCloud云盘同步频繁出错，10KB大小的文件同步几分钟，提示“No file”同时无法打开任何文件。慎用。

插件篇

External Links: 自动发现潜在链接的插件

该官方插件 (Core Plugin) 有两个功能：(1) Links: 罗列出当前文档中包含的指向其他文档的链接；(2) Unlinked Mentions: 罗列出当前文档的正文中提及的其他文档的名字，并询问是否插入链接。

如何开启

Settings -> Core Plugins -> Open External Links。开启后最右侧工具栏上方会多出一个Outgoing Link图标。

写到这里想到，如果用NLP model做一个基于语义的链接发现插件可能会很有趣。

Dictionary: 简单美丽的词典插件

上图是不是看起来像一个词典界面呢！其实它是一个一键（三键）生成的markdown文档。在用Obsidian做笔记的过程中遇到生词，只需要一秒钟就可以获得一个集音标、释义、例句、同义词、词源、甚至mp3于一体的生词卡片，并保存在指定目录下，之后可以回顾整理、导入Anki、导出csv或者任意引用。

如何开启

  1. Settings -> Community plugins -> Browse -> 搜索 "Dictionary" -> Install. （如果开启了Safe Mode需要先将其关闭：Settings -> Community plugins -> Safe Mode -> Turn off）
  2. 安装后Settings主界面下方 PLUGIN OPTIONS 列表会出现一个新标签 Dictionary。点击该标签，开启Synonym Suggestions和Show Options in Context Menu. 还可以进行一系列自定义设置，包括多语种、保存路径、排版方式等。
  3. 回到主界面，最右侧工具栏上方会多出一个词典图标。

如何使用

  1. 以上图为例，在左侧文档"Writing a paper step by step"标题下，选中单词"fleeting"(若已开启Synonym Suggestions, 会自动显示其同义词)。若已开启Show Options in Context Menu，单击右键 -> 选择Look up, 最右侧工具栏会出现词典解释。
  2. 点击工具栏上方New Note按钮，即可将词典解释保存成markdown文档。开启预览模式即可看到图中右侧文档效果。
  3. 给左侧文档中的原单词加上[[]] (fleeting -> [[fleeting]])，选择弹窗中建议的链接，即可将该单词链接到右侧文档。
